# 전기차 충전소 분포·접근성 대시보드

대한민국 공공데이터를 결합해 전기차 충전 인프라의 **지리적 분포**와 **수요 대비 공급 부족 지역**을 보여주는 Streamlit 대시보드.

## 지표

| ID | 지표 | 단위 | 공식 |
|---|---|---|---|
| **M1** | EV 1,000대당 충전기 수 | **시도 (17)** | `충전기수 / (전기차등록수 / 1000)` |
| **M2** | 인구 10만명당 충전기 수 | **시군구 (~230)** | `충전기수 / (인구 / 100000)` |
| M3 | 급속충전기 비율 | 시도·시군구 | `급속수 / 전체수` |
| M4 | 충전소당 충전기 수 | 시도·시군구 | `충전기수 / 충전소수` |
| M5 | 가용률 (스냅샷) | 시도·시군구 | `사용가능 / (사용가능 + 충전중)` |

M1·M2 모두 **값이 낮을수록 부족**하다. 오름차순 정렬이 곧 부족 지역 랭킹이다.

**M1이 시도 단위인 이유:** 한국전력공사 전기차 등록 데이터가 17개 시도로만 제공된다. 시군구로 쪼갤 방법이 없어 해상도를 지표별로 분리했다.

## 설치

```bash
uv venv .venv --python 3.12          # 또는 python3 -m venv .venv
uv pip install --python .venv/bin/python -r requirements.lock
```

`requirements.lock`은 해시가 고정된 잠금 파일이다. 재현 가능한 설치는 항상 이걸 쓴다.
`requirements.txt`는 직접 의존성만 적힌 입력 파일이며, 의존성을 바꿀 때만 편집하고 아래로 잠금 파일을 다시 만든다.

```bash
uv pip compile requirements.txt --python .venv/bin/python \
    --generate-hashes --output-file requirements.lock
```

인증키를 설정한다. 공공데이터포털([data.go.kr](https://www.data.go.kr))에서 "한국환경공단 전기자동차 충전소 정보" 서비스를 신청해 받는다.

```bash
cp .env.example .env
# .env 를 열어 EV_API_KEY 에 발급받은 키를 넣는다
```

## 실행

```bash
python src/collect.py     # ① API 전수 수집 -> data/raw/chargers_YYYYMMDD.parquet
python src/clean.py       # ② 정제 -> data/processed/chargers_clean.parquet
python src/metrics.py     # ③ 지표 산출 -> data/processed/metrics_*.parquet
streamlit run src/app.py  # ④ 대시보드
```

대시보드는 **API를 호출하지 않는다.** Parquet 스냅샷만 읽으므로 항상 빠르고 쿼터를 쓰지 않는다.

### API 쿼터 주의

`getChargerInfo`는 **일 1,000회** 제한이 있다. 전수 수집은 53회(`numOfRows=9999`)면 끝나므로 여유롭지만, `collect.py`에는 페이징 버그로 쿼터를 태우는 사고를 막는 `MAX_CALLS = 80` 하드 가드가 있다.

같은 날짜 스냅샷이 이미 있으면 `collect.py`는 **건너뛴다.** 강제로 다시 받으려면 `--force`를 준다.

`getChargerStatus`는 **호출하지 않는다.** `getChargerInfo` 응답이 이미 `stat`(충전기 상태)과 `statUpdDt`(상태갱신일시)를 포함하기 때문이다.

## 인구 데이터 (M2 해상도)

M2는 **있는 인구 파일 중 가장 높은 해상도**로 자동 계산된다.

| `data/raw/` 의 파일 | M2 해상도 | 상태 |
|---|---|---|
| `jumin_sgg_202606.csv` | **시군구 230개** | 권장 |
| `jumin_sido_202606.csv` | 시도 17개 | 현재 사용 중 |
| 둘 다 없음 | M2 skip | 나머지 지표·탭은 정상 동작 |

둘 다 있으면 시군구가 이긴다. 파일을 넣고 `python src/metrics.py`를 다시 실행하면 대시보드가 자동으로 올라간다.

[jumin.mois.go.kr](https://jumin.mois.go.kr) → 주민등록 인구 및 세대현황 → **월간**:

| 설정 | 값 |
|---|---|
| 조회 기간 | **2026년 06월 ~ 2026년 06월** (단월. M2는 스냅샷 지표라 한 달이면 충분) |
| 등록구분 | 거주자 |
| 행정구역 | 전체 선택 후 **시군구까지 펼침** ← 유일하게 실패할 수 있는 지점 |

> 시도를 접은 채로 받으면 17행짜리 파일이 나온다. 받고 나서 **행 수가 250줄 안팎인지** 확인하라. 20줄이면 시도 파일이다.

쓰는 컬럼은 `행정구역`과 `..._총인구수` 둘뿐이다. 세대수·남녀 인구는 있어도 무시한다.

## 참조데이터 재생성

`data/ref/`의 두 CSV는 OpenAPI 활용가이드(.docx)의 공통코드 표에서 생성한다. 가이드가 개정되면 다시 만든다.

```bash
python scripts/build_ref.py --docx /path/to/활용가이드_v1.23.docx
```

- `zscode_map.csv` — 시군구 코드 230개 (`zscode` → 시도/시군구명)
- `sido_name_map.csv` — 한전 축약 시도명 17개 → `zcode`

`sido_name_map.csv`가 필요한 이유: 한전 CSV는 `강원`·`전북`을 쓰고 행정구역 정식명은 `강원특별자치도`·`전북특별자치도`다. 매핑이 어긋나면 **에러 없이 시도 하나가 통째로 누락**된다. `tests/test_ref.py`가 17개 전부 매칭되는지 검사한다.

## 전남광주통합특별시 (zcode 12) 처리

2026년 API 응답에 **신설 광역단체 `전남광주통합특별시`(zcode 12)** 가 등장한다. 기존 광주광역시(29) + 전라남도(46)의 27개 시군구가 `12xxx` 코드로 이관됐다 (충전기 31,962기).

문제는 **참조 데이터가 둘 다 아직 분리 체계**라는 것이다.

| 데이터 | 기준일 | 체계 |
|---|---|---|
| 충전소 API | 2026-07 | **통합** (zcode 12) |
| 한전 전기차 등록 | 2025-12-31 | 분리 (광주 / 전남) |
| 주민등록 인구 | 2026-06 | 분리 (광주광역시 / 전라남도) |

통합 코드를 그대로 쓰면 M1(EV 대비)과 M2(인구 대비) 조인이 **둘 다 조용히 깨진다.** 그래서 `clean.py`가 API의 신규 코드를 **레거시 17개 시도 체계로 되돌린다.** 27개 시군구가 이름 기준 1:1로 대응하므로 정보 손실은 없다.

```bash
python scripts/build_bridge.py   # data/ref/zscode_bridge.csv 생성 (27행)
```

브리지는 최신 스냅샷에서 자동 생성되며, 1:1 대응이 깨지면 즉시 실패한다. `tests/test_region_bridge.py`가 이를 검증한다.

> 결과적으로 대시보드는 광주와 전남을 **분리해서** 보여준다. 행정구역은 통합됐지만 비교 대상인 전기차·인구 데이터가 아직 분리 집계이기 때문이다. 참조 데이터가 통합 체계로 갱신되면 이 브리지를 제거하면 된다.

## 데이터 함정

구현하면서 확인한, 조용히 데이터를 오염시키는 것들:

- **`delYn == 'Y'` 행이 API 응답에 섞여 온다.** 활용가이드에 "최근 삭제된 충전기 정보 제공"이라 명시돼 있다. 필터하지 않으면 충전기 수가 과대집계되는데, 에러 없이 그럴듯한 숫자가 나와 알아채기 어렵다.
- **결측이 JSON `null`이 아니라 문자열 `"null"`로 온다.** 그대로 두면 결측 처리가 안 된다.
- **좌표가 범위를 벗어난 행이 있다.** `clean.py`가 위도 33.0~38.7 / 경도 124.5~131.9로 검증하고 `coord_valid` 비율이 98% 미만이면 FAIL을 로그에 찍는다. 무효 좌표 충전기는 **지도에서만 빠지고 집계에는 포함**된다.
- **지도에 52만 포인트를 개별 마커로 찍으면 브라우저가 멈춘다.** folium 마커는 피했지만, pydeck에 원본 포인트를 그대로 넘겨도 **브라우저로 가는 JSON이 40.8MB**가 되어 같은 문제가 재발한다. `display.py`의 `grid_aggregate()`가 서버에서 약 2km 격자로 미리 묶어 **8,652셀 / 0.82MB (50배 축소)** 로 보낸다.
- **`stat` 코드표는 필드 설명이 아니라 공통코드 3.2절이 정본이다.** `getChargerInfo` 필드 설명은 `0~5`만 적어놓았지만, 코드표에는 **`6`(예약중)과 `9`(상태미확인)** 가 더 있다. 실제 API는 `9`를 19,897기 보낸다. `.map()`으로 라벨링하면 미매칭이 NaN이 되고 `value_counts()`가 조용히 빼버려 **표 합계가 KPI와 2만기 어긋난다.** `label_stat()`이 미정의 코드도 라벨을 갖게 해서 막고, 테스트는 **실데이터가 아니라 공식 코드표**를 기준으로 검증한다 — 실데이터 기준이면 지금 0건인 `6`이 빠져도 그냥 통과하기 때문이다.
- **세종특별자치시는 시군구가 없는 유일한 광역단체다.** 주민등록 CSV에 `세종특별자치시 (3600000000)` 시도 행 하나로만 나오는데, 충전소 API는 세종 충전기에 `zscode=36110`을 준다. 시도 행이라고 버리면 **충전기 6,474기가 M2에서 NaN이 되어 접근성 랭킹에서 통째로 사라진다.** `load_population()`이 세종만 예외 처리한다.
- **인구 결측률은 시군구 "행 수"가 아니라 "충전기 수"로 가중해서 잰다.** 행 기준이면 세종이 통째로 빠져도 `1/230 = 0.4%`로 보여 5% 게이트를 그냥 통과한다. 충전기 기준으로는 1.25%다.
- **필터를 반영한 컬럼과 아닌 컬럼을 한 표에 섞으면 안 된다.** 지표 정의는 `metrics.aggregate_region()` 한 곳에만 두고, 대시보드는 필터된 프레임에 그 함수를 다시 적용한다. 안 그러면 "급속만" 필터에서 급속 비율이 26%로 표시된다.

## 테스트

```bash
python -m pytest tests/ -v          # 단위 + 통합 (107개)
python scripts/smoke_app.py         # 대시보드 본문 실행 + 렌더 예산(5초) 확인
```

스모크는 `streamlit run` 이 아니라 스크립트 본문을 직접 실행한다. Streamlit 은 정적 셸을 HTTP 200 으로 먼저 주고 본문은 웹소켓 연결 후에 돌리기 때문에, **HTTP 200 은 `app.py` 가 실제로 렌더된다는 증거가 되지 못한다.**

## 기준시점

세 데이터의 기준일이 다르다. 대시보드 상단에 각주로 표시된다.

| 데이터 | 기준일 |
|---|---|
| 충전기 | 수집 스냅샷 시점 |
| 전기차 등록 (한전) | 2025-12-31 |
| 인구 (주민등록) | 2026-06 |
