# 프로젝트 컨텍스트

## 목적

대한민국 공공데이터를 결합해 전기차 충전 인프라의 지리적 분포와 수요 대비 공급 격차를 보여주는 분석 대시보드다.

## 도메인 용어

- **충전기:** 하나의 충전 커넥터 단위. `charger_count`의 대상이다.
- **충전소:** 동일한 `statId`로 묶이는 물리적 장소. `station_count`의 대상이다.
- **충전기 스냅샷:** 특정 수집 시점의 충전기 상태와 위치를 보존한 Parquet 데이터.
- **부족 지역:** M1 또는 M2 값이 낮아 수요 대비 공급이 취약한 지역.
- **미진출 지역:** 운영기관 필터에서 선택 기관의 충전기가 0개인 지역. 국가 전체 인프라 부족과 구분한다.
- **M1:** 시도별 전기차 1,000대당 충전기 수. 낮을수록 부족하다.
- **M2:** 가능한 최고 행정 해상도의 인구 10만명당 충전기 수. 낮을수록 접근성이 취약하다.
- **M3:** 전체 충전기 중 급속충전기 비율.
- **M4:** 충전소당 충전기 수.
- **M5:** 충전대기 상태가 충전대기와 충전중의 합에서 차지하는 비율. UI에서는 엄밀한 의미를 설명한다.
- **희소 집계:** 원본 51만 건을 브라우저로 보내지 않고 필터 차원별로 미리 계산한 화면용 데이터.
- **Split Command:** 승인된 React 화면 구조. 어두운 고정 필터 레일과 밝은 분석 캔버스를 분리한다.

## 경계

- Python 수집·정제·지표 파이프라인이 데이터 정의의 유일한 원본이다.
- 제품 화면은 React 단일 페이지 UI다. 기능 동등성 검증 후 기존 Streamlit UI는 제거됐다.
- 첫 React 버전은 로컬 HTTP에서 정적 희소 집계를 읽으며 백엔드 API를 두지 않는다.
- 외부 배경지도 타일 통신은 허용하지만 충전 데이터 API 호출과 명확히 구분한다.

## 피해야 할 용어 혼용

- 특정 운영기관의 `미진출`을 국가 인프라 `부족`으로 부르지 않는다.
- 시도 단위 M1과 시군구 단위 M2를 같은 해상도로 설명하지 않는다.
- 배치 스냅샷을 `실시간` 데이터라고 부르지 않는다.
