# 디자인

## 기준 문서
- 상태: 활성
- 최종 갱신: 2026-07-23
- 현재 제품 화면: 로컬 HTTP에서 실행하는 React 단일 페이지 분석 대시보드 (`prototype/`)
- 이전 화면: Streamlit 대시보드(`src/app.py`)는 React 기능 동등성 검증 후 제거됐다 (2026-07-13)
- 검토 근거: `README.md`, 승인 스펙, 구현 계획, `src/codebook.py`, 테스트, Obsidian UI/UX 기록

## 권장안
- **권장 UI 스택:** React + Vite + TypeScript
- **차트:** Apache ECharts
- **지도:** MapLibre GL JS
- **스타일:** CSS Modules와 CSS 사용자 정의 속성. 초기 단계에서는 별도 디자인 시스템 패키지를 만들지 않는다.
- **데이터 공급:** Python 빌드 단계가 화면용 희소 집계 파일을 생성하고 React가 정적 파일만 읽는다. 초기 버전에는 FastAPI를 도입하지 않는다.
- **전환 방식:** Python 수집·정제·지표 계산은 유지하고, 승인된 한 화면 프로토타입을 기반으로 Streamlit 표시 계층을 React 로 교체했다. 구조 승인(2026-07-13) 이후의 React 코드는 폐기용이 아니라 운영 품질로 작성했고, Streamlit 표시 계층은 제거됐다.
- **선정 이유:** 지도 상호작용, 반응형 배치, URL 상태 공유, 접근성, 애니메이션과 시각 회귀 테스트를 Streamlit보다 세밀하게 통제할 수 있다.
- **승인된 화면 구조:** 밝은 토스형 분석 작업공간. 좌측 유틸리티 필터 레일과 넓은 분석 캔버스를 분리하고, 상단 KPI 아래에 지도와 취약 지역 랭킹을 나란히 배치한다.

## 브랜드
- 성격: 차분하고 명료한 분석 도구. 토스의 기능적 블루와 간결한 계층을 사용하되 공공 인프라 데이터의 정확성을 우선한다.
- 신뢰 신호: 기준일, 단위, 지표 공식, 결측 및 해상도 한계를 화면 가까이에 노출한다.
- 피해야 할 것: 의미 없는 그라데이션, 과도한 둥근 카드, 장식용 아이콘, 숨겨진 데이터 주의사항. 밝은 표면은 블루 액션과 명확한 데이터 계층을 동반해야 한다.

## 제품 목표
- 목표: 전국 충전 용량, 지리적 편중, 수요 대비 부족 지역을 수초 안에 파악하게 한다.
- 목표: 필터 변경이 지도·차트·표에 동시에 반영되게 한다.
- 비목표: 충전기 실시간 제어, 실시간 API 폴링, 회원 기능, 마케팅 랜딩 페이지.
- 성공 신호: 첫 화면에서 핵심 KPI와 취약 지역을 확인할 수 있다.
- 성공 신호: 사용자가 전기차 기준(시도)과 인구 기준(시군구)의 서로 다른 행정 해상도를 혼동하지 않는다.
- 성공 신호: 전국 지도와 초기 데이터가 일반적인 데스크톱 환경에서 5초 이내 표시된다.

## 사용자와 작업
- 주요 사용자: 공공정책 분석가, 충전 인프라 기획자, 데이터에 익숙한 일반 사용자.
- 사용자 작업: 공급과 수요 비교, 지역 격차 탐색, 운영기관·속도·24시간 조건별 분석, 데이터 한계 확인.
- 주요 이용 환경: 데스크톱·태블릿 분석 우선, 모바일은 핵심 KPI와 순위를 읽을 수 있는 수준으로 지원.

## 정보 구조
- 주 화면: 별도 화면 전환이 없는 단일 페이지 `Split Command`다. 분석 캔버스는 KPI → 충전기 상태 분포 → 지도와 취약 지역 랭킹의 병렬 패널 순서로 읽힌다.
- 기능 동등성 마이그레이션: 기존 Streamlit의 네 탭(실제 라벨 `개요`, `분포 지도`, `부족 지역 랭킹`, `접근성 랭킹`; 기존 문서의 작업명 `전국 현황`, `분포 지도`, `부족 지역`, `접근성`)은 2026-07-13 React 전환 때 위 패널들로 흡수됐다. 네 화면을 별도 내비게이션으로 복원하지 않는다.
- 데스크톱: A `Split Command` 구조를 기준으로 고정 좌측 필터 레일과 우측 분석 캔버스를 사용한다.
- 모바일: 상단 요약과 핵심 KPI·순위를 세로로 제공하며 정교한 지도 조작과 전용 필터 시트는 후속 단계로 둔다.
- 콘텐츠 순서: 데이터 기준일 → 활성 필터 → 핵심 KPI → 주 시각화 → 상세 표 → 해석 주의사항.
- 첫 버전 범위: 기존 네 화면의 분석 작업을 한 화면 패널로 흡수하고 네 필터의 기능 동등성을 구현한다. 후속 기능은 `백로그`에서 명시적으로 관리한다.

## 디자인 원칙
- 장식보다 신호: 토스 블루는 주요 행동과 선택에만 쓰고, 시각적 강조는 분석 중요도에 따라 배치한다.
- 소비자 금융 앱이 아닌 전력 인프라: 전력망, 좌표 격자, 신호선 문법을 사용한다.
- 비교 가능성 우선: 같은 지표는 모든 화면에서 같은 색, 단위, 소수점 규칙을 사용한다.
- 주의사항을 숨기지 않음: 기준일 불일치, 무효 좌표, 낮은 해상도를 관련 시각화 옆에 표시한다.
- 점진적 공개: 첫 화면은 결론을, 상세 표와 도구 설명은 검증 근거를 제공한다.
- 절충: 분석 밀도를 유지하되 작은 화면에서는 비교보다 읽기 순서를 우선한다.

## 시각 언어
- 색상: 캔버스는 `#FFFFFF`, 조용한 표면은 `#F2F4F6`, 주요 행동·선택은 `#3182F6`, 강조 정보는 `#1B64DA`, 취약은 앰버, 오류는 `#E42939`로 고정한다.
- 의미 색상: 선택과 주요 행동은 블루, 취약은 앰버, 오류는 적색으로 고정한다.
- 글꼴: `Toss Product Sans`를 우선 지정하고, 설치·배포 권한이 확인되지 않은 상태에서는 self-host 하지 않는다. 현재 번들에는 `IBM Plex Sans KR` 시스템 대체 글꼴을 포함하며 외부 CDN 호출은 금지한다.
- 간격: 8px 기준 리듬, 12열 데스크톱 그리드, 최대 콘텐츠 폭 1440px.
- 형태: 반경 8~16px, `#E5E8EB` 1px 테두리, 그림자보다 평면적인 표면 계층.
- 움직임: 첫 진입과 필터 결과 갱신에만 150~400ms 전환을 사용한다.
- 이미지와 아이콘: 스톡 이미지는 사용하지 않는다. 기능 아이콘은 하나의 SVG 아이콘 집합으로 제한한다.

## 컴포넌트
- 이 절의 이름은 파일 단위 컴포넌트 목록이 아니라 **논리적 화면 영역**을 뜻한다. `AppShell`, `DataBasisBar`, `FilterRail`, `ActiveScope`, `MetricCard`, `RankingTable`, `DataCaveat`는 `App.tsx` 안의 JSX 영역이며 별도 컴포넌트 파일이 아니다.
- 구현 경계: `App.tsx`는 화면 상태와 렌더링을 조정하고, `selectors.ts`는 랭킹·지도 취약도·활성 필터 설명·0건 복구 가능성·상태 분포·운영기관 개수 같은 순수 도메인 파생을 담당한다. 희소 집계는 `data.ts`, Python이 내보낸 지표 정의의 평가는 `metrics.ts`가 담당한다.
- `AppShell`(논리 영역): 좌측 필터와 주 분석 영역을 소유한다.
- `DataBasisBar`(논리 영역): 충전기·전기차·인구 기준일과 해상도를 항상 표시한다.
- `FilterRail`(논리 영역): 네 가지 필터, 초기화, 선택 결과 수를 제공한다. 운영기관은 검색 가능한 복수 선택과 상위 10개 빠른 선택을 제공한다.
- `ActiveScope`(논리 영역): 현재 필터를 제거 가능한 칩으로 표시한다.
- `MetricCard`(논리 영역): 값, 단위, 정의와 지표별 주의사항을 표시한다.
- `DistributionMap`(별도 컴포넌트): 시군구 경계 코로플레스를 기본으로 제공하고, 2km 격자를 시군구 내부 편중을 드러내는 보조 오버레이로 겹친다. 충전기 밀도는 히트맵과 밀집 원(비례 원)으로도 전환해 탐색한다. 히트맵은 인접 셀의 강도를 연속 면으로 연결해 전국의 밀집 축을 빠르게 찾게 한다. 밀집 원은 색을 정보 시안으로 고정하고 원 크기만 충전기 수에 비례시켜 부족(앰버)과 밀집(시안)의 시각 신호를 분리한다. 또한 격자 셀 중심점만 사용하므로 시군구 경계가 없어도 동작하고 배경지도 로드 실패 시에도 원을 유지한다. 3D 기둥은 ADR 0001의 여섯 제약(기본값 아님, 색·높이 이중 부호화, bearing 0 고정과 pitch 45도 상한, 무필터 전국 기준 고정 높이 도메인, 전국 축척에서 비활성, 자기 고지 문구)을 모두 만족할 때만 보조 모드로 허용한다. 기둥 높이는 공급량이 아니라 **취약도**에 매핑한다 — 높이를 충전기 수에 매핑하면 "높이=많음"이 되어 이 화면의 결론("어디가 부족한가")과 정반대 신호를 준다.
- `DataProvenance`(별도 컴포넌트): 필터 레일 하단에서 프로젝트 목적, 세 데이터 출처(한국환경공단 충전소 OpenAPI, 한국전력 지역별 전기차, 행정안전부 주민등록 인구), 각 출처가 어느 지표의 분자·분모인지, 기준일, 이용허락범위를 밝힌다. 문구의 단일 출처는 `docs/data-sources.md` 이며 확인되지 않은 라이선스 유형을 추정해 적지 않는다.
- 지표 극성(polarity): 각 지표는 "낮을수록 취약"인지 "높을수록 취약"인지를 정의와 함께 한 곳에 고정한다. 색 방향·랭킹 방향·3D 높이 방향이 모두 여기서 파생되어야 같은 지표가 모든 화면에서 같은 색을 갖는다.
- `RankingChart`(별도 컴포넌트): 취약 방향을 고정하고 아래 표와 같은 파생 배열을 시각화한다. 막대 선택에 따른 교차 필터링은 아직 구현하지 않았다.
- `RankingTable`(논리 영역): 차트와 같은 취약 지역 배열을 단위에 맞게 포맷해 제공한다. 정렬·검색·CSV 내보내기는 아직 구현하지 않았다.
- `DataCaveat`(논리 영역): 결측, 기준일 차이, 지도 제외 수를 문맥 안에서 설명한다.
- 상태 변형: 기본, 선택, 로딩, 비어 있음, 경고, 오류, 비활성.
- 토큰 소유권: 프런트엔드 루트 CSS 변수가 유일한 색상·간격·타이포그래피 원본이다.

## 접근성
- 목표 기준: WCAG 2.1 AA.
- 키보드: 모든 필터, 지도·랭킹 표시 방식 버튼, 차트 대체 표, 지도 외 핵심 정보에 키보드로 접근할 수 있어야 한다.
- 초점: 블루 2px 외곽선을 사용하고 배경과 충분한 대비를 확보한다.
- 색상: 취약·선택·오류를 색상만으로 전달하지 않고 텍스트와 패턴을 병행한다.
- 스크린 리더: 제목 구조, 폼 레이블, 표 캡션, 상태 갱신용 `aria-live`를 제공한다.
- 움직임: `prefers-reduced-motion`에서 비필수 애니메이션을 제거한다.
- 지도 대체: 지도와 동일한 집계를 표 또는 순위 목록으로 제공한다.

## 반응형 동작
- 지원 범위: 데스크톱 1280px 이상, 태블릿 768px 이상, 모바일 360px 이상.
- 데스크톱: 어두운 좌측 필터 280px, 밝은 데이터 캔버스, KPI 4열, 지도 중심의 넓은 화면.
- 태블릿: 필터를 상단 가로 영역으로 이동하고 KPI를 2열로 배치한다.
- 모바일: KPI와 순위를 1열로 쌓아 읽을 수 있게 하되 첫 버전에는 전용 필터 시트와 전체 지도 조작 동등성을 요구하지 않는다.
- 터치: 최소 터치 영역 44px, hover에만 의존하는 정보는 금지한다.
- 차트: 모바일에서는 축 레이블을 줄이고 상세값은 아래 표에서 제공한다.

## 상호작용 상태
- URL 분석 상태 공유: 시도·속도·운영기관 이름·24시간 필터와 지도·랭킹 토글을 쿼리에 저장하며, 공유 링크와 새로고침에서 복원하고 유효하지 않은 값은 차원별 기본값으로 안전하게 강등한다.
- 로딩: 셸은 즉시 표시하고 KPI에는 개별 스켈레톤을 사용한다. 상태 분포·지도·랭킹은 정적 집계 로드 후 함께 표시한다.
- 비어 있음: 어떤 필터가 결과를 0건으로 만들었는지 표시하고 해당 조건을 한 번에 해제할 수 있게 한다.
- 오류: 데이터 종류, 실패 시점, 재시도 방법을 한국어로 안내하되 내부 예외는 노출하지 않는다.
- 성공: 활성 스냅샷 기준일과 필터 후 충전기 수를 표시한다.
- 비활성: 이유를 화면의 설명 문구 또는 도구 설명으로 함께 제공한다.
- 느린 연결: 정적 셸과 KPI를 먼저, 외부 배경지도 타일과 지도 레이어를 나중에 로드한다.
- 정적 파일 오류: 어떤 집계 파일이 없거나 손상됐는지 표시하고 재생성 명령을 안내한다.

## 콘텐츠 문체
- 문체: 직설적이고 기술적으로 정확한 한국어.
- 용어: 화면에서는 전기차 기준·인구 기준·급속 비율처럼 뜻을 먼저 쓰고, M1~M5는 데이터 계약과 문서의 보조 식별자로만 사용한다. 시도·시군구 단위는 `README.md`와 동일하게 유지한다.
- 숫자: 개수는 천 단위 구분, 비율은 소수점 한 자리, 기준일은 `YYYY-MM-DD`.
- 미세문구: 랭킹은 “낮을수록 취약”을 반복 표시하고 단위와 모집단을 생략하지 않는다.
- 금지: “실시간”처럼 실제 데이터 계약보다 강한 표현, 근거 없는 가치 판단, “가용률”과 “유휴율”의 혼용.

## 구현 제약
- 프런트엔드: React + Vite + TypeScript를 사용한다. Next.js의 서버 렌더링은 현재 로컬 분석 제품에 필요하지 않으므로 도입하지 않는다.
- 실행 범위: 첫 버전은 로컬 HTTP 서버만 지원한다. `file://` 직접 실행과 공개 정적 호스팅은 범위에서 제외한다.
- 백엔드: 첫 버전에는 FastAPI를 도입하지 않는다. Python이 필터 차원별 희소 집계 파일을 생성하며, 용량 또는 5초 성능 예산을 충족하지 못할 때만 읽기 전용 API를 재검토한다.
- 데이터 경계: 기존 `clean.py`와 `metrics.py`가 지표의 유일한 원본이다. 프런트엔드는 합산 공식을 복제하지 않는다.
- 성능: 52만 원본 좌표를 브라우저로 보내지 않는다. Python이 지역 지표와 지도 격자를 운영기관·속도·24시간 차원별 희소 집계 파일로 생성한다.
- 지도 통신: 충전 데이터는 로컬 정적 파일만 사용하지만 외부 배경지도 리소스 통신은 허용한다. 이 범주는 기존 문서에서 말하는 배경지도 타일과 MapLibre가 타일을 찾기 위해 먼저 읽는 스타일 메타데이터를 포함한다. 충전 데이터와 배경지도 통신 경계를 UI와 문서에서 구분한다.
- 호환성: 기존 네 필터, 레거시 네 화면에서 제공하던 분석 작업, M1~M5, 단위 배지, 기준일 각주를 한 화면 패널 구조에서 유지한다.
- 의존성: 실제 구현 계획 승인 전에는 패키지를 추가하지 않는다.
- 프로토타입: 실제 집계 데이터 일부로 헤더, 기준일, 필터, KPI 4개, 지도, 취약 지역 랭킹을 포함한 한 화면을 만들고 1440px·768px에서 승인받는다. **2026-07-13 승인 완료.** 구조 승인 이후의 모든 코드는 폐기용이 아니라 **운영 품질**로 작성한다 — 테스트, 접근성, 에러 상태, 성능 예산을 처음부터 만족시킨다.
- 지도 기하: 시군구 경계는 단순화한 TopoJSON 을 `data/ref/` 에 커밋해 쓴다(목표 600KB 미만). 원본 SHP 는 `data/raw/` 에 두고 커밋하지 않는다. 공식 2025 SGIS 원본의 고유 물리 경계 229개와 현행 코드가 정확히 일치하는지 계약 테스트로 강제한다. `zscode_map.csv`의 폐지 군위 코드 `47720`은 현행 `27720`의 별칭으로만 조인하며 도형을 복제하지 않는다. 경계 파일이 없으면 지도는 격자 단독으로 우아하게 강등되고, 강등 사실을 화면이 밝힌다. 근거와 획득 절차는 ADR 0001.
- 검증: 정적 데이터 계약 테스트, Vitest 컴포넌트 테스트, Playwright 핵심 흐름, axe 접근성 검사, 데스크톱·태블릿 스크린샷 회귀를 요구한다.
- 마이그레이션: React 화면이 기존 Python 테스트와 데이터 정합성 스팟체크, Playwright E2E 를 모두 통과해 Streamlit 표시 계층을 제거했다 (2026-07-13).

## 구현 완료 기준
- 한 화면의 KPI·상태 분포·지도·랭킹 패널과 네 필터가 (제거된) 기존 Streamlit 네 화면의 분석 작업과 기능적으로 동등하다.
- 지도는 집계 데이터만 사용하고 초기 표시가 5초 이내다.
- M1 최하위 지역의 손계산 결과가 API와 화면 값에 일치한다.
- 768px와 1440px 기준 화면에서 가로 스크롤이나 잘린 핵심 정보가 없으며 360px에서 KPI와 순위를 읽을 수 있다.
- 키보드만으로 필터 변경, 패널 순서 탐색, 지도·랭킹 표시 방식 전환, 순위 확인이 가능하다.
- Python 테스트, 프런트엔드 테스트, 접근성 검사, 시각 회귀가 모두 통과한다.
