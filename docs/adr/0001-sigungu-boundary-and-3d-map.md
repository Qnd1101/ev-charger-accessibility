# ADR 0001. 시군구 경계 코로플레스 도입과 3D 기둥 조건부 허용

- 상태: 제안
- 날짜: 2026-07-13
- 관련: `DESIGN.md` 68행(3D 금지), 132행(코로플레스 열린 질문), 114행(통신 경계)

## 맥락

React 프로토타입의 지도는 2km 격자 셀만 그린다. 전국 축척에서는 셀이 점처럼 흩어져
"지도"가 아니라 "산점도"로 읽힌다. 지역 단위 결론(어느 시군구가 부족한가)을 지도가
전달하지 못하고, 랭킹 표만 그 일을 하고 있다.

동시에 제품 책임자가 입체 표현을 요구했다. `DESIGN.md` 68행은 3D 기둥을 명시적으로
금지하고 있으므로, 도입하려면 금지의 근거를 검토하고 조건을 명문화해야 한다.

## 결정

### 1. 시군구 경계 코로플레스를 지도의 기본 표현으로 채택한다

`DESIGN.md` 132행의 열린 질문("시군구 인구 CSV 확보 후 코로플레스를 2단계 범위에
포함할지")을 **채택**으로 닫는다. 시군구 인구는 확보됐고(`scripts/build_jumin.py`),
M2가 시군구 해상도로 산출되므로 코로플레스의 분모가 존재한다.

2km 격자는 삭제하지 않고 **보조 오버레이**로 강등한다. 격자는 코로플레스가 보여주지
못하는 시군구 **내부**의 편중(도심 집중, 외곽 공백)을 드러내며, 코로플레스 계급의
근거 역할을 한다.

### 2. 3D 기둥을 조건부로 허용하되, 높이는 공급량이 아니라 **취약도**에 매핑한다

금지 조항의 근거는 타당했다. 높이는 직관적으로 "많음"을 신호하는데, 이 지도의 결론은
"부족함"이다. 충전기 수를 그대로 돌출시키면 서울·경기가 솟고 정작 봐야 할 부족 지역이
바닥에 깔린다 — **결론과 정반대 방향의 시각 신호**다.

따라서 높이를 뒤집는다. **M2가 낮을수록 높게** 솟는다. 부족한 지역이 융기하므로 높이와
결론의 방향이 일치한다.

3D는 다음 여섯 제약을 **전부** 만족할 때만 허용한다. 하나라도 빠지면 68행의 원래 금지가
되살아난다.

1. **기본값이 아니다.** 마운트 시 항상 2D 코로플레스이고, 필터를 바꾸면 3D는 해제된다.
2. **이중 부호화.** 높이와 색이 같은 값(M2 취약도)을 인코딩한다. 기둥이 앞 기둥에 가려도
   색이 값을 유지하므로 정보가 소실되지 않는다.
3. **카메라 구속.** bearing 0 고정, pitch 상한 45도. 회전으로 특정 지역을 크게 보이게
   만들 수 없다.
4. **고정 높이 도메인.** 높이 스케일의 최대값은 *무필터 전국* 기준으로 고정한다. 필터를
   좁혔다고 기둥이 자라면 화면 간 비교가 깨진다.
5. **줌 게이트.** 전국 축척에서는 기둥 겹침이 심하므로 3D 토글을 비활성화하고 사유를
   툴팁으로 밝힌다(`DESIGN.md` 97행 비활성 상태 변형).
6. **자기 고지.** 3D 활성 시 지도가 "높이는 탐색용입니다. 정확한 비교는 순위 표를
   사용하세요"를 스스로 표시한다.

`prefers-reduced-motion` 및 모바일에서는 3D를 제공하지 않는다.

### 3. 경계 데이터는 국토교통부 시군구 경계를 쓰고, 단순화 산출물만 `data/ref/`에 커밋한다

**후보 조사 결과 커뮤니티 데이터(`southkorea/southkorea-maps`)는 쓸 수 없다.** 실측 결과
해당 데이터의 코드 251개와 우리 `zscode` 230개의 **교집합이 0개**다. 2013년 옛 통계청
코드 체계이며 청주 통합(2014) 등 이후 개편도 반영되지 않았다.

국토교통부 시군구 경계(data.go.kr, `SIG_CD` 필드)는 법정동코드 앞 5자리를 쓰므로 우리
`zscode`와 같은 체계다.

**반드시 행정구역 개편 이전 시점 파일을 받아야 한다.** 개편 이후 파일에는 전남광주통합
특별시 코드(`12xxx`)가 들어 있는데, 이를 광주(29)·전남(46)으로 되돌리는 역매핑이
저장소에 없다. 인천은 `src/regions.py`의 브리지로 병합 가능하지만 광주·전남은 불가능하다.

용량은 단순화로 해결한다. 원본 SHP 수십 MB → mapshaper 단순화 + TopoJSON 인코딩으로
**300~600KB** 목표(같은 계열 데이터 실측 기준 GeoJSON 55MB → TopoJSON 2.09MB, 약 26배).
`DESIGN.md` 113행의 "원본을 브라우저로 보내지 않는다"와 5초 예산을 지킨다.

커밋 정책:
- 원본 SHP는 `data/raw/` (커밋 금지, `.gitignore` 기존 규칙)
- 단순화 산출물 `data/ref/sigungu.topo.json`은 **커밋한다.** 파생 지표와 달리 이건 변하지
  않는 참조 좌표계이고, data.go.kr이 로그인을 요구해 CI 자동 재생성이 불가능하다.
- 커밋 전에 **공공누리 유형을 원 페이지에서 눈으로 확인**하고 출처·기준일·유형을
  `data/ref/sigungu.topo.json.LICENSE`에 명시한다. 라이선스 확인 전에는 커밋하지 않는다.

계약 테스트로 조인을 강제한다: 경계의 코드 집합과 `zscode_map.csv`의 230개가 **양방향으로
정확히 일치**해야 하고(빠짐도 잉여도 0), 파일 크기 상한과 좌표 범위(WGS84 한반도)를 함께
단언한다. 단순화 시 `keep-shapes`를 빼면 옹진군·울릉군 같은 소면적 폴리곤이 소멸해 230개
카운트가 깨지므로, 이 테스트가 그것을 잡는다.

## 대안과 기각 사유

- **3D를 계속 금지한다.** 코로플레스 + 취약 상위 라임 발광 테두리 + 격자 오버레이만으로도
  "지도답고 인상적"이라는 요구는 상당 부분 충족된다. 그러나 제품 책임자가 입체 표현을
  명시적으로 요구했고, 위 여섯 제약이 왜곡 위험을 실질적으로 제거하므로 조건부 허용을 택한다.
- **deck.gl 오버레이(HexagonLayer/ColumnLayer).** 시각적으로 가장 화려하지만 `DESIGN.md`
  116행이 승인 전 패키지 추가를 금지하고, 번들이 크다. MapLibre 내장 `fill-extrusion`으로
  충분하다 — 새 의존성이 0이다.
- **레이어를 선언적으로 구성하는 범용 지도 컴포넌트.** 네 화면이 한 컴포넌트를 재사용한다는
  이점이 있으나, 데이터 계약상 존재할 수 없는 조합이 다수 열린다(`grid_cube`에는 `zscode`가
  없어 격자에 M2를 얹는 경로가 원천적으로 불가능하다). 과잉 일반화로 판단해 기각한다.
  다만 "지표마다 취약 방향(polarity)을 한 곳에 고정한다"는 아이디어는 채택해
  `DESIGN.md` 48행("같은 지표는 모든 화면에서 같은 색")을 구조적으로 보장한다.

## 결과

- `DESIGN.md` 68행을 개정한다(3D 조건부 허용, 코로플레스 기본, 격자 오버레이 강등).
- `DESIGN.md` 132행 열린 질문을 닫는다.
- 경계 데이터 확보가 코로플레스 구현의 선행 조건이 된다. 확보 전까지 지도는 격자 단독으로
  **우아하게 강등**되어야 하며, 강등 경로를 1급 상태로 설계한다.
