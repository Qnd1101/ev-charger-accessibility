# ADR 0002. 제품 의도 정본과 화면 런타임 통신 경계

- 상태: 채택
- 날짜: 2026-07-22
- 관련: `CONTEXT.md`, `DESIGN.md`, `README.md`, ADR 0001

## 맥락

외부 아키텍처 리뷰 입력이 이 프로젝트의 "원래 의도"를 오프라인 우선 목업 대시보드로
설명했다. 그 입력은 네트워크와 지도 SDK를 쓰지 않고 CSV 픽스처와 `Coordinate Preview`를
사용한다고 전제했다. 그러나 저장소의 거버넌스 문서와 구현 이력은 실제 공공데이터 수집,
지도 기반 분포 탐색, 로컬 정적 집계를 일관되게 전제한다.

제품 의도를 외부 입력의 회고적 주장과 저장소 문서 중 어디에서 판정할지, 그리고 "로컬
데이터"가 수집 파이프라인까지 오프라인이라는 뜻인지 화면 런타임의 통신 경계를 뜻하는지
명확히 고정할 필요가 있다.

## 이력 검증

Git 전체 이력의 유일한 루트이자 최초 커밋은
`931a6b2ef534f3507c3722863c060efbfeba7308`(2026-07-13,
`전기차 충전소 접근성 대시보드 초기 구현`)이다. 부모 커밋은 없다. 커밋 본문은 다음과
같이 제품을 설명한다.

> 공공데이터 3종(충전소 API, 한전 전기차 등록, 주민등록 인구)을 결합해
> 충전 인프라의 지리적 분포와 수요 대비 공급 부족 지역을 보여주는
> Streamlit 대시보드.

같은 최초 커밋에는 다음 증거가 함께 들어 있다.

- `src/collect.py`: `ENDPOINT = "https://apis.data.go.kr/B552584/EvCharger/getChargerInfo"`와
  전수 수집 함수 `collect()`가 있다.
- `README.md`: `python src/collect.py`로 API 전수 수집 후 `streamlit run src/app.py`로
  대시보드를 실행하도록 안내한다.
- `src/app.py`: `pydeck`을 임포트하고 `st.pydeck_chart(...)`로 지도 레이어를 그리며,
  개요·분포 지도·부족 지역·접근성 탭을 제공한다.

결정 시점에 `git rev-list --all`로 확인 가능한 전체 커밋을 검사한 결과 모든 커밋에 위 실제
API 엔드포인트를 가진 수집기가 있었고, 모든 커밋에 `pydeck` 또는 `maplibre-gl` 지도 SDK가
있었다. 따라서 이 저장소의 Git 이력에는 오프라인 전용·지도 SDK 금지·CSV 픽스처 전용·
`Coordinate Preview` 조합이 제품 의도를 지배한 단계가 없다. 그 의도 목록은 확인 가능한
어떤 저장소 상태와도 일치하지 않는다.

최초 Streamlit 화면도 **화면 렌더 중에는** 충전 데이터 API를 호출하지 않고 수집된 Parquet
스냅샷을 읽었다. 이는 전체 프로젝트가 오프라인 목업이었다는 증거가 아니라, 수집과 화면
런타임을 분리한 현재 통신 경계의 초기 형태다.

## 결정

### 1. 저장소 거버넌스 문서 집합을 제품 의도의 단일 정본으로 삼는다

`CONTEXT.md`, `DESIGN.md`, 채택된 `docs/adr/` 문서가 함께 제품 의도의 유일한 권위 있는
출처를 이룬다.

- `CONTEXT.md`는 목적, 도메인 용어, 시스템 경계를 정의한다.
- `DESIGN.md`는 현재 화면 구조, 사용자 경험, 구현 제약을 정의한다.
- ADR은 중요한 결정의 맥락, 변경 이유, 채택·기각한 대안을 기록한다.

`README.md`는 설치·운영 가이드이고 코드와 테스트는 현재 동작의 증거다. 이들은 정본 문서와
일치해야 하지만, 외부 리뷰 프롬프트나 과거 구현 단편과 마찬가지로 제품 의도를 독자적으로
재정의하지 않는다. 정본 사이에 충돌이 생기면 가장 최근의 채택 ADR로 결정을 확인하고 같은
변경에서 `CONTEXT.md`와 `DESIGN.md`를 함께 정합화한다.

### 2. 오프라인 목업 제약은 이 저장소를 지배한 적이 없는 것으로 기록한다

최초 커밋부터 실제 공공데이터 API 수집기와 Streamlit 지도 SDK가 존재했다. 2026-07-13
React 기능 동등성 마이그레이션은 이 제품 의도를 폐기하지 않고, 표시 계층을 MapLibre GL과
정적 희소 집계 기반의 단일 페이지로 교체했다. 따라서 "네트워크 없음", "지도 SDK 없음",
"CSV 픽스처 전용", "Coordinate Preview"는 과거 요구사항이나 복원 대상이 아니다.

### 3. 화면 런타임의 통신 경계를 유지한다

충전 데이터는 Python 수집·정제·지표 파이프라인이 미리 만든 로컬 정적 희소 집계 파일로만
화면에 공급한다. React 화면은 공공데이터 API를 직접 호출하지 않으며 지표 공식을 재구현하지
않는다. 화면 런타임에서 허용하는 외부 통신은 배경지도 리소스뿐이다. 여기서 기존 문서의
"배경지도 타일"은 타일 자체와 MapLibre가 타일을 찾기 위해 먼저 읽는 스타일 메타데이터를
포괄하는 통신 경계의 이름이다. 배경지도 실패 시에도 로컬 집계와 경계 비의존 지도 표현은
읽을 수 있게 강등한다.

이 경계는 배치 수집기 `src/collect.py`가 공공데이터 API와 통신하는 것을 금지하지 않는다.
수집 단계의 외부 API 통신과 화면 런타임의 로컬 충전 데이터 소비를 구분한다.

## 대안과 기각 사유

- **외부 리뷰 입력을 숨은 원래 의도로 우선한다.** 저장소의 최초 상태와 전체 이력에 반하고,
  검증할 수 없는 출처가 거버넌스 문서를 덮어쓰게 되므로 기각한다.
- **현재 제품을 오프라인 CSV 목업으로 되돌린다.** 실제 수집·지표 파이프라인과 지도 기반
  분석 목표를 제거하며 승인된 `Split Command` 구조에도 반하므로 기각한다.
- **React 화면이 공공데이터 API를 직접 호출한다.** API 쿼터, 재현성, 기준일 고정, 지표의
  단일 원본 원칙을 깨므로 기각한다.
- **외부 통신을 모두 금지해 배경지도도 제거한다.** 지리적 맥락을 불필요하게 약화한다.
  충전 데이터와 배경지도 통신은 분리할 수 있고 실패 시 강등 경로도 있으므로 기각한다.

## 결과

- 제품 의도 충돌은 `CONTEXT.md` + `DESIGN.md` + 채택 ADR을 기준으로 판정한다.
- 외부 입력이 다른 "원래 의도"를 주장하면 Git 이력과 정본 문서로 검증하고, 근거가 없으면
  저장소 요구사항으로 수용하지 않는다.
- 충전 데이터는 화면에서 로컬 정적 파일만 사용하고, 외부 통신은 배경지도 리소스에 한정한다.
- 제품 의도를 바꾸는 후속 결정은 새 ADR과 관련 정본 문서 갱신을 같은 변경으로 남긴다.
