# 데이터 출처와 저작권

화면(`DataProvenance` 패널)과 README가 함께 참조하는 **단일 출처**다. 출처 문구를 화면에
직접 적지 말고 이 문서를 갱신한 뒤 반영한다.

## 1. 한국환경공단 전기자동차 충전소 정보 (OpenAPI)

- 링크: https://www.data.go.kr/data/15076352/openapi.do
- 엔드포인트: `https://apis.data.go.kr/B552584/EvCharger/getChargerInfo` (`src/collect.py`)
- 제공기관: 한국환경공단
- **이용허락범위: 공공누리 제1유형 (출처표시)** — 2026-07-13 원 페이지 확인
- 포털 수정일: 2026-04-20
- 우리 스냅샷: `data/raw/chargers_20260712.parquet` (충전기 517,417기)

**무엇에 쓰는가:** 화면의 모든 충전기 집계의 원본이다. 충전기 1기가 1행이며, 여기서
`statId`(충전소), `zscode`(시군구), `busiNm`(운영기관), `chgerType`(급속/완속),
`useTime`(24시간 여부), `stat`(실시간 상태), 위경도를 쓴다. M1·M2·M3·M4·M5가 전부 이
데이터에서 나온다.

**왜 표준데이터셋(`15013115` 전국전기차충전소표준데이터)이 아닌가:** 표준데이터셋에는
충전기 실시간 상태(`stat`)가 없어 M5 유휴율을 만들 수 없다.

## 2. 한국전력공사 지역별 전기차 현황정보

- 링크: https://www.data.go.kr/data/15039554/fileData.do
- 제공기관: 한국전력공사
- **이용허락범위: 제한 없음** — 2026-07-13 원 페이지 확인. 공공누리 유형 표기는 없다.
- 포털 수정일: 2026-06-17
- 우리 파일: `data/raw/kepco_ev_20251231.csv` (기준일 2025-12-31)

**무엇에 쓰는가:** **M1(충전기/EV 1,000대)의 분모**다. 시도 단위 전기차 등록 대수만
제공하므로 M1의 해상도가 시도로 고정된다. M2가 시군구인데 M1이 시도인 이유가 이것이며,
화면은 이 해상도 차이를 배지로 밝혀야 한다.

## 3. 행정안전부 주민등록 인구통계

- 링크: https://jumin.mois.go.kr/
- 제공기관: 행정안전부 주민과
- **이용허락범위: 확인되지 않음.** 2026-07-13 기준 사이트에 공공누리 유형 표기가 없고
  하단에 `ⓒ Ministry of the Interior and Safety. All rights reserved.` 만 있다.
  **화면에 공공누리 유형을 적지 마라 —** 확인되지 않은 사실이다. "행정안전부 주민등록
  인구통계"라는 출처 표시와 링크까지만 적는다.
- 우리 파일: `data/raw/jumin_sgg_202606.csv` (기준월 2026-06)

**무엇에 쓰는가:** **M2(충전기/인구 10만명)의 분모**다. 이 사이트는 시군구까지 펼치면
한 번에 한 시도씩만 내려주므로, 17개 조각 파일을 `data/raw/jumin_parts/` 에 두고
`scripts/build_jumin.py` 가 하나로 병합한다. 병합 시 일반구(수원시 장안구 등 39행)와
세종 중복 행을 제외한다 — 그대로 두면 인구가 이중 계상돼 M2가 조용히 틀린다.

## 4. 시군구 행정경계 (코로플레스용, 미확보)

- 후보: 국토교통부 시군구 경계 (data.go.kr / 국가공간정보포털)
- 상태: **미확보.** ADR 0001 참조.
- **커밋 전에 원 페이지에서 이용허락범위를 눈으로 확인하고 이 문서에 기록한다.**

## 표기 규칙

- 화면과 문서 모두 **확인된 것만** 적는다. 공공누리 유형이 확인되지 않은 출처에
  "제1유형" 따위를 추정으로 적지 않는다.
- 각 출처 옆에 **기준일**을 함께 표시한다. 세 출처의 기준일이 서로 다르다는 사실 자체가
  해석상 중요한 주의사항이다(충전기 2026-07-12, 전기차 2025-12-31, 인구 2026-06).
- 배경지도 타일(CARTO Positron)은 데이터 출처가 아니라 **외부 통신 경계**다. 충전 데이터는
  전부 로컬 정적 파일이고 배경지도만 외부에서 온다는 사실을 화면이 구분해 밝힌다
  (`DESIGN.md` 114행).
