# 전기차 충전 인프라 접근성 분석

## 대학 과제 요약

이 프로젝트는 대한민국 전기차 충전 인프라가 지역별 수요에 비해 어떻게 분포하는지 분석하는 대학 과제입니다. 공공데이터를 결합해 단순 충전기 수뿐 아니라 전기차 수와 인구를 함께 비교하고, 상대적으로 부족할 수 있는 지역을 찾습니다.

## 연구 질문

1. 전기차 등록 대수에 비해 충전기가 적은 시도는 어디인가?
2. 인구에 비해 충전기가 적은 시군구는 어디인가?
3. 급속 충전기 비율과 수집 시점의 충전기 상태는 지역별로 어떻게 다른가?

## 데이터와 방법

| 데이터 | 제공기관 | 분석에서의 역할 |
|---|---|---|
| 전기자동차 충전소 정보 | 한국환경공단 | 충전기 수, 충전소, 급속 여부, 위치, 수집 시점 상태 |
| 지역별 전기차 현황 | 한국전력공사 | 전기차 기준의 분모 |
| 주민등록 인구통계 | 행정안전부 | 인구 기준의 분모 |
| 행정구역 경계 | 국가데이터처 SGIS | 시군구 단위 지도 표시 |

Python 파이프라인이 원본을 수집·정제·집계하고, React 대시보드는 미리 만든 집계 JSON만 읽습니다. 따라서 브라우저는 원본 충전기 목록을 다시 계산하거나 외부 충전 데이터 API를 호출하지 않습니다.

## 비교 기준 읽는 법

| 화면 이름 | 계산 | 낮은 값의 뜻 |
|---|---|---|
| 전기차 기준 | 충전기 수 ÷ (전기차 등록 대수 ÷ 1,000) | 전기차 수에 비해 충전기가 적을 수 있음 |
| 인구 기준 | 충전기 수 ÷ (인구 ÷ 100,000) | 주민 수에 비해 충전기가 적을 수 있음 |
| 급속 비율 | 급속 충전기 수 ÷ 전체 충전기 수 | 급속 충전기의 비중이 작음 |
| 충전소 밀도 | 충전기 수 ÷ 충전소 수 | 한 충전소에서 이용할 수 있는 충전기 수가 적음 |
| 현재 비어 있음 | 충전대기 충전기 수 ÷ 응답한 충전기 수 | 수집 시점에 비어 있는 충전기의 비중 |

## 해석의 한계

- 데이터마다 기준일이 다르므로 같은 날의 완전한 실시간 비교가 아닙니다.
- 전기차 등록 대수는 시도 단위라 전기차 기준도 시도 단위로만 비교합니다.
- 인구 기준은 인구 파일의 해상도에 따라 시군구 또는 시도로 계산될 수 있습니다.
- 충전기 상태는 수집 시점의 스냅샷입니다. 실제 고장 여부나 대기 시간까지 뜻하지 않습니다.
- 낮은 지표 값은 추가 조사가 필요한 신호이며, 그 자체로 정책 우선순위를 확정하지 않습니다.

## 재현 방법

```bash
python src/clean.py
python src/metrics.py
python scripts/build_web_data.py
cd prototype
npm run dev
```
