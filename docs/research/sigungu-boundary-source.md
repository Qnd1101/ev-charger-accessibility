# 시군구 경계 공식 자료 선행 조사

- 조사일: 2026-07-13
- 범위: `data/ref/zscode_map.csv`의 개편 전 시군구 코드 230개와 결합할 경계 자료
- 결론: **인증 없이 공식 2025-06-30 SGIS 경계 원본을 확보하고 변환할 수 있다.** 저장소의
  230개 데이터 코드에는 현행 대구 군위 `27720`과 폐지된 경북 군위 `47720`이 동시에
  들어 있어 공식 물리 경계는 229개다. 최종 TopoJSON은 중복 없는 229개 경계를 유지하고,
  화면 경계 어댑터가 군위 코드 별칭을 명시적으로 처리한다.

## 확보 원본: 2025-06-30 SGIS 행정구역 통계 및 경계

국가데이터처의 [SGIS 행정구역 통계 및 경계][sgis-data-go]는 2025년 기준 행정구역 경계를
SHP로 제공한다. 공공데이터포털 원문은 제공기관을 국가데이터처 공간정보서비스과,
이용허락범위를 **제한 없음**, 제공형태를 공공데이터포털 원문파일 다운로드로 표시한다.
페이지의 구조화 메타데이터에는 다음 비로그인 직접 다운로드 URL이 있으며, 2026-07-13
HEAD 요청에서 HTTP 200, `Content-Length: 251294559`, ZIP 첨부파일 응답을 확인했다.

`https://www.data.go.kr/cmm/cmm/fileDownload.do?atchFileId=FILE_000000003601705&fileDetailSn=1&insertDataPrcus=N`

확보한 `data/raw/sgis_admin_boundaries_2025.zip`은 같은 251,294,559 bytes이고 SHA-256은
`3F517984BFDF4BBE43EE2A8849CFF010D70AC5A826F880E6976B9A1F2B30611B`이다. 원본은
252개 레코드이며 DBF의 `BASE_DATE`는 모두 `20250630`, 코드 필드는 `SIGUNGU_CD`,
`.cpg`는 UTF-8이다. `.prj`는 Korea 2000 / Unified CS(EPSG:5179)를 선언한다.

세종은 SGIS 코드 `29010` 한 건으로 존재하며 명시적 매핑을 통해 저장소 코드 `36110`으로
변환된다. 일반구를 상위 시 코드로 dissolve하면 공식 원본의 서로 다른 물리 경계는 229개다.

### 230코드 계약의 군위 중복

`zscode_map.csv`에는 현행 대구 군위 `27720`과 2023년 7월 1일 대구 편입 전에 쓰던 경북
군위 `47720`이 함께 있다. 그러나 SGIS 2025 원본에는 현행 군위 `22520` 도형 한 건만 있고,
실제 충전기 스냅샷도 `27720` 203기, `47720` 0기다. `regions.json`의 `47720` 인구도
결측이다.

최종 `sigungu.topo.json`은 공식 물리 경계인 `27720`만 유지한다. `47720`은 화면 경계
어댑터에서 `27720`으로 정규화하며, 두 코드에 유효한 값이 동시에 있으면 모호한 조인으로
거부한다. 따라서 동일 위치의 no-data 피처가 현행 `27720` 값을 덮지 않는다.

출처, 정확한 변환 명령, 해시와 이 제한은 `data/ref/sigungu.topo.json.LICENSE`에 기록했다.

## 역사 시점 보조 후보: 2026-06-09 일별법정구역정보 전체데이터

국토교통부의 [일별법정구역정보 공공데이터포털 원문][daily-data-go]은 시도·시군구·읍면동
등 법정구역 경계를 월별·일별로 제공하며, 행정구역 변경 이력을 추적할 수 있는 SHP
자료라고 설명한다. 같은 페이지는 제공기관을 국토교통부 국가공간정보센터, 제공 경로를
[VWorld 법정구역정보 데이터셋][daily-vworld], 이용허락범위를 **제한 없음**으로 표시한다.

VWorld에서 기준일을 2026-06-01~2026-06-30으로 조회하면 **기준일 2026-06-09,
전체데이터, 전국, SHP, 약 124MB** 항목이 나온다. 이 날짜는 인천 행정체제 개편 시행일인
2026-07-01보다 앞선다. 개편 시행일과 구 체계 변경은 행정안전부의
[인천 행정체제 개편 보도자료][incheon-mois] 및 [설치 법률][incheon-law]에서 확인된다.
따라서 저장소의 개편 전 인천 코드와 맞출 수 있는 가장 구체적인 공식 시점 후보다.

다만 다음 사항은 **파일을 내려받기 전에는 확인할 수 없다.**

- SHP 속성에서 시군구 레벨을 식별하는 필드와 코드값
- 일반구를 포함한 원본 코드 집합을 상위 시로 합친 뒤 `zscode_map.csv` 230개와 정확히
  일치하는지
- DBF 문자 인코딩과 `.cpg` 파일 존재 여부
- 도형 오류, 도서 지역 보존 여부와 단순화 후 파일 크기

VWorld 원문은 배포 좌표계를 **GRS80 / EPSG:5186**, 포맷을 SHP, 공급 방식을 웹 다운로드
및 API 서비스로 표시한다. 브라우저용 산출물은 EPSG:4326(WGS84)으로 재투영해야 한다.

### 다운로드 절차

1. VWorld에 로그인한다. 비로그인 상태에서 다운로드 버튼은 “로그인 후 이용” 안내만
   표시한다.
2. [법정구역정보 데이터셋][daily-vworld]에서 기준일을 `2026-06-01`~`2026-06-30`으로
   제한한다.
3. `기준일 2026-06-09`, `구분 전체데이터`, `전국`, `SHP` 항목을 선택해 내려받는다.
4. 압축을 해제한 원본은 커밋하지 않고 `data/raw/` 아래에 둔다.
5. `.cpg`가 있으면 그 값을 기록하고, 없으면 `ogrinfo`로 한글 필드를 확인해 인코딩을
   판정한다. 인코딩을 추정해 변환하지 않는다.

## 보조 후보: 2025 센서스 시군구 경계

[국토교통부 센서스 시군구 경계 원문][census-data-go]은 기준 파일명을
`국토교통부_(센서스경계)시군구경계_20250120`, 포맷을 SHP, 전체 행을 250개, 제공기관을
국토교통부 국가공간정보센터, 이용허락범위를 **제한 없음**으로 표시한다. 원 제공 경로는
[VWorld 센서스 시군구 경계][census-vworld]이며, VWorld 메타정보는 좌표계를
**GRS80 / EPSG:5186**, 갱신주기를 연간으로 표시한다.

저장소 목표가 230개인데 원문 행 수가 250개인 이유는 일반구가 있는 10개 시를 상위 시와
일반구로 동시에 세는 체계와 일치하는 것으로 보인다. 저장소에는 수원·성남·안양·안산·고양·
용인·청주·천안·포항·창원이 상위 시 코드 한 개씩으로 들어 있다. **250에서 일반구 30개를
10개 상위 시 도형으로 합치면 230개가 된다는 것은 현재 단계의 추론이며, 실제 코드와 도형을
확인하기 전에는 채택 근거가 아니다.**

또한 VWorld의 현재 센서스 파일은 2026-04-10 갱신본으로 교체되어 있고 기준일이 표시되지
않는다. 공공데이터포털의 2025 메타데이터와 현재 다운로드 파일이 동일하다고 볼 수 없으므로,
2025 파일을 별도로 확보하지 못하면 이 후보를 사용하지 않는다.

## 사용하지 않을 후보

[VWorld 시군구 2D 데이터 API][vworld-api]는 `sig_cd`, `full_nm`, `sig_kor_nm`과 GeoJSON
응답, EPSG:4326을 공식 지원하므로 기술적으로는 편리하다. 그러나 원문 갱신일이
2026-07-06이고 인천 개편 이후 자료다. 또한 이 API의 이용허락 표시는
[공공데이터포털 API 원문][api-data-go]에서 **공공누리 제1유형 및 제3자 권리 포함**으로
표시되어, 위 파일데이터의 “제한 없음”과 동일한 라이선스로 간주할 수 없다. 이번 개편 전
경계 자산의 원본으로 사용하지 않는다.

커뮤니티 저장소와 비공식 변환물은 조사·채택 대상에서 제외했다.

## 확보 후 필수 계약 검증

원본을 받으면 다음 순서로 검증한다. 도구 명령은 예시이며, 실제 필드명은 `ogrinfo` 결과를
근거로 확정한다.

1. `ogrinfo -so -al <원본.shp>`로 CRS, 필드, feature 수, geometry type을 기록한다.
2. 시군구 도형만 선택하고 일반구는 명시적인 매핑표로 상위 10개 시에 합친다. 코드 앞자리를
   임의로 자르는 방식은 쓰지 않는다.
3. 재투영한다: `ogr2ogr -t_srs EPSG:4326 ...`.
4. `mapshaper`에서 `keep-shapes`를 사용해 단순화하고 TopoJSON으로 내보낸다.
5. 경계 코드와 `data/ref/zscode_map.csv`에서 폐지 군위 `47720`을 제외한 현행 코드 집합을
   양방향 비교해 **누락 0, 잉여 0, 총 229개**를 단언한다.
6. 모든 좌표가 저장소의 대한민국 범위(경도 124.5~131.9, 위도 33.0~38.7)에 있고,
   `data/ref/sigungu.topo.json`이 600KB 미만인지 확인한다.
7. 옹진군·울릉군·제주 도서부 등 작은 도형이 남았는지 육안 및 계약 테스트로 확인한다.
8. 결함 주입으로 코드 하나 삭제, 잉여 코드 하나 추가, 좌표계 미변환, `keep-shapes` 누락 시
   테스트가 실제 실패하는지 확인한다.

검증을 모두 통과한 경우에만 단순화 산출물과 함께 원문 URL, 기준일 2025-06-30,
이용허락범위 제한 없음, 변환 절차를 `data/ref/sigungu.topo.json.LICENSE`에 기록한다.

## 확인 사실과 미해결 사항

### 확인 사실

- 저장소 `zscode_map.csv`는 230개이며 개편 전 인천·광주·전남 코드를 사용한다.
- 인증 없는 공식 SGIS 2025-06-30 원본 다운로드가 가능하고, 원본 ZIP과 응답 크기가
  251,294,559 bytes로 일치한다.
- SGIS 원본은 252개 레코드, UTF-8, EPSG:5179이며 세종 코드 `29010`을 포함한다.
- 공식 일별법정구역정보에 개편 전인 2026-06-09 전국 전체 SHP가 존재한다.
- 일별법정구역정보와 2025 센서스 경계의 공공데이터포털 이용허락범위는 제한 없음이다.
- 두 VWorld 데이터셋의 배포 CRS는 EPSG:5186이다.

### 추론

- 2026-06-09 전체본은 법정구역의 특정 시점 재현이 필요할 때 가장 적합한 원본 후보다.
- 센서스 경계 250개를 일반구 기준으로 dissolve하면 목표 230개가 될 가능성이 높다.

### 미해결

- VWorld 로그인 부재로 2026-06-09 일별법정구역 SHP 자체는 확보하지 못했다.
- 최종 TopoJSON은 서로 다른 공식 물리 경계 229개와 용량·좌표·도서 도형 검증을 통과한다.
