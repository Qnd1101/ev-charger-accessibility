# 도메인 문서 사용 규칙

이 저장소는 단일 컨텍스트 구조를 사용한다.

## 탐색 전에 읽을 문서

- 루트 `CONTEXT.md`
- 작업 영역과 관련된 `docs/adr/` 문서
- UI 작업은 루트 `DESIGN.md`

파일이 아직 없으면 작업을 중단하지 않는다. 새로운 도메인 용어나 되돌리기 어려운 결정이 생길 때 `/domain-modeling`으로 보완한다.

## 용어 규칙

- 이슈, 스펙, 테스트 이름은 `CONTEXT.md`의 용어를 사용한다.
- 같은 개념에 새로운 동의어를 만들지 않는다.
- 기존 ADR과 충돌하는 제안은 충돌 사실과 재검토 이유를 명시한다.
